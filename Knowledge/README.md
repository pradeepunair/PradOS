# Knowledge

Durable reference material across all projects and time periods.

## Structure

```
Knowledge/
├── Reference/      ← Facts, preferences, frameworks that don't expire
│   └── about-me.md
├── Research/       ← Deep-dives on topics (fintech, AI, competitors)
└── People/         ← Notes on individuals: stakeholders, collaborators
```

## Adding to Knowledge

- **Reference/**: Timeless facts. Update in-place.
- **Research/**: Topic-based files. Name: `YYYY-MM-DD-[topic].md`
- **People/**: One file per person. Name: `[first-last].md`

## What belongs here vs. Projects/

- `Projects/` = context for a specific initiative (expires when project ends)
- `Knowledge/` = context that applies across projects (never expires)
