# Projects

Each project lives in its own folder: `Projects/[project-name]/`

## Structure of Each Project

```
Projects/[project-name]/
├── brief.md          ← Start here. Problem, goals, constraints, stakeholders
├── research/         ← Raw inputs: articles, data, competitive analysis
└── assets/           ← Outputs: decks, diagrams, exported files
```

## Creating a New Project

Run `/new-project [name]` or copy the `_template/` folder.
Always fill in `brief.md` before doing any other work.
